# 1. 프로젝트 개요

이번 프로젝트는 여러 종류의 API가 하나의 서비스 안에서 어떤 역할을 하는지 보여주기 위한 수업용 예제입니다.

수업에서 다루는 API는 각각 따로 존재하는 기능처럼 보일 수 있습니다. 하지만 실제 서비스에서는 로그인, 데이터 조회, 지도 표시, 결제 기능이 하나의 사용자 흐름 안에서 연결됩니다.

이 프로젝트에서는 아래 흐름을 하나의 미니 서비스로 구성합니다.

```text
네이버 로그인
↓
공공 API 데이터 조회
↓
네이버 지도에 데이터 표시
↓
관심 지역 구독 상품 확인
↓
나이스페이먼츠 테스트 결제
↓
결제 결과 확인
```

이 수업의 핵심은 완성형 상용 서비스를 만드는 것이 아닙니다.

수강생이 **“API는 내 서비스에 외부 기능을 연결하는 방법이구나”**라는 감각을 갖게 하는 것이 목표입니다.

# 2. 서비스 컨셉

## 2-1. 서비스명

**공공데이터 지도 구독 서비스**

---

## 2-2. 한 줄 설명

사용자가 네이버로 로그인한 뒤 공공데이터를 조회하고, 조회된 정보를 네이버 지도에 표시하며, 관심 지역 정보 구독을 테스트 결제로 체험하는 웹서비스입니다.

---

## 2-3. 서비스 예시 시나리오

사용자는 사이트에 접속해 네이버 로그인을 합니다.

로그인 후 지역명 또는 키워드를 입력해 공공데이터를 조회합니다.

조회된 데이터는 목록으로 표시되고, 위치 정보가 있는 데이터는 네이버 지도 위에 마커로 표시됩니다.

사용자는 관심 지역 데이터를 계속 받아보기 위해 구독 상품을 확인하고 테스트 결제를 진행합니다.

결제가 완료되면 결제 성공 화면에서 구독 상태를 확인합니다.

# 3. 사용자 흐름

사용자 입장에서 보는 전체 흐름은 아래와 같습니다.

1. 사이트 접속
2. 네이버 로그인 버튼 클릭
3. 로그인 완료 후 대시보드 이동
4. 지역 또는 키워드 입력
5. 공공데이터 조회
6. 조회 결과 목록 확인
7. 지도에서 마커 확인
8. 관심 지역 구독 상품 확인
9. 테스트 결제 진행
10. 결제 성공 또는 실패 결과 확인

---

## 3-1. API별 역할

| API 종류 | 역할 |
|---|---|
| 네이버 로그인 API | 사용자가 누구인지 확인 |
| 공공 API | 외부 데이터를 가져오기 |
| 네이버 지도 API | 데이터를 위치 정보로 보여주기 |
| 나이스페이먼츠 API | 구독 결제 흐름 만들기 |
| Supabase | 사용자, 구독, 결제 데이터 저장 |

# 5. 주요 화면 구성

---

## 5-1. 랜딩 및 로그인 화면

### 목적

사용자가 서비스의 목적을 이해하고 네이버 로그인을 통해 입장하게 합니다.

### 화면 구성

- 서비스명
- 간단한 서비스 설명
- 네이버 로그인 버튼
- 수업용 안내 문구

### 예시 문구

```text
공공데이터 지도 구독 서비스

네이버로 로그인하고,
공공데이터를 지도에서 확인해보세요.
```

---

## 5-2. 대시보드 화면

### 목적

로그인한 사용자가 공공데이터를 조회할 수 있게 합니다.

### 화면 구성

- 사용자 환영 문구
- 지역 또는 키워드 입력 영역
- 공공데이터 조회 버튼
- 조회 결과 요약
- 지도 화면으로 이동하는 버튼 또는 지도 영역

### 예시 문구

```text
안녕하세요, 홍길동님.
조회하고 싶은 지역 또는 키워드를 입력해보세요.
```

---

## 5-3. 데이터 목록 화면

### 목적

공공 API로 가져온 데이터를 사용자가 이해하기 쉬운 목록 형태로 보여줍니다.

### 표시 항목 예시

- 시설명 또는 데이터명
- 주소
- 전화번호
- 운영 여부
- 데이터 출처
- 지도 표시 가능 여부

---

## 5-4. 지도 화면

### 목적

공공 API 데이터 중 위치 정보가 있는 항목을 네이버 지도에 표시합니다.

### 화면 구성

- 지도 영역
- 데이터 목록
- 마커
- 마커 클릭 시 정보창

### 중요 포인트

공공 API 데이터에 좌표가 없을 수도 있습니다.

이 경우 주소를 좌표로 변환하는 과정이 필요할 수 있습니다.

수업에서는 좌표가 있는 데이터만 우선 표시하거나, 주소 → 좌표 변환은 개념으로 설명합니다.

---

## 5-5. 구독 화면

### 목적

API로 만든 서비스가 어떻게 수익화 흐름으로 이어지는지 보여줍니다.

### 화면 구성

- 구독 상품 카드
- 가격
- 제공 기능
- 테스트 결제 버튼

### 예시 상품

```text
관심 지역 알림 구독

월 9,900원

제공 기능:
- 관심 지역 데이터 모아보기
- 신규 공공정보 알림
- 즐겨찾기 저장
- 상세 검색 기능
```

---

## 5-6. 결제 결과 화면

### 목적

결제 성공 또는 실패 결과를 사용자에게 보여줍니다.

### 화면 구성

- 결제 성공/실패 메시지
- 결제 금액
- 상품명
- 구독 상태
- 메인으로 돌아가기 버튼

# 9. AI 프롬프트 작성법

---

## 9-1. API 연동 프롬프트의 핵심

AI에게 API 연동을 요청할 때는 **“API 붙여줘”**라고만 말하면 부족합니다.

아래 정보를 함께 전달해야 결과가 좋아집니다.

- 어떤 API를 쓰는지
- 이 API의 역할이 무엇인지
- 사용자가 어떤 행동을 하면 API가 호출되는지
- 요청에 필요한 값은 무엇인지
- 응답에서 어떤 데이터를 사용할 것인지
- 화면에는 어떻게 보여줄 것인지
- API Key나 Secret은 어디에 보관할 것인지
- 에러가 나면 어떻게 처리할 것인지

---

## 9-2. 전체 프로젝트 기획 프롬프트

```text
Next.js와 Supabase를 사용해서 수업용 공공데이터 지도 구독 서비스의 기본 뼈대를 만들고 싶어.

서비스 흐름:
1. 네이버 로그인
2. 공공 API 데이터 조회
3. 네이버 지도에 데이터 표시
4. 구독 상품 표시
5. 나이스페이먼츠 테스트 결제
6. 결제 성공/실패 화면 표시

이번 단계에서는 API Key 발급이나 개발자 콘솔 설정은 제외하고,
서비스 화면 구조, 페이지 구성, 데이터 흐름, 데이터 테이블 초안, 개발 Phase를 먼저 만들어줘.

초보자도 이해할 수 있도록 각 화면의 목적, 필요한 API, 사용자 행동, 결과를 정리해줘.
```

---

## 9-3. 네이버 로그인 프롬프트

```text
Next.js 프로젝트에서 네이버 로그인을 연결하고 싶어.

목표:
- 사용자가 네이버 로그인 버튼을 누르면 네이버 로그인 페이지로 이동한다.
- 로그인 성공 후 콜백 페이지로 돌아온다.
- 서버에서 인증 코드를 Access Token으로 교환한다.
- Access Token으로 사용자 정보를 조회한다.
- 사용자 이름과 이메일을 Supabase users 테이블에 저장하거나 업데이트한다.
- 로그인 완료 후 대시보드 페이지로 이동한다.

주의사항:
- Client Secret은 브라우저에 노출되지 않게 서버 쪽에서만 사용한다.
- 환경변수 이름도 함께 제안해줘.
- 초보자가 이해할 수 있도록 단계별로 설명해줘.
```

---

## 9-4. 공공 API 프롬프트

```text
Next.js 프로젝트에서 공공 API 데이터를 조회하는 기능을 만들고 싶어.

목표:
- 사용자가 지역명 또는 검색어를 입력한다.
- 조회 버튼을 누르면 서버에서 공공 API를 호출한다.
- 응답 데이터에서 이름, 주소, 전화번호, 좌표 정보를 추출한다.
- 결과를 카드 목록으로 보여준다.
- 좌표가 없는 데이터는 지도 표시 대상에서 제외한다.

주의사항:
- API Key는 환경변수로 관리한다.
- 공공 API 응답 필드명이 일정하지 않을 수 있으니 데이터 정리 함수를 따로 만들어줘.
- 에러가 발생하면 사용자에게 안내 메시지를 보여줘.
```

---

## 9-5. 네이버 지도 프롬프트

```text
Next.js 프로젝트에서 네이버 지도 API를 사용해 공공 API 데이터를 지도에 표시하고 싶어.

목표:
- 지도 화면에 네이버 지도를 표시한다.
- 공공 API에서 가져온 데이터 중 latitude, longitude 값이 있는 항목만 지도 마커로 표시한다.
- 마커를 클릭하면 시설명, 주소, 전화번호가 정보창에 보인다.
- 왼쪽에는 목록, 오른쪽에는 지도가 보이는 레이아웃으로 구성한다.
- 목록을 클릭하면 해당 마커 위치로 지도가 이동한다.

주의사항:
- 지도 API Key는 환경변수로 관리한다.
- 지도 스크립트가 로드되기 전에는 로딩 상태를 보여준다.
- 좌표가 없는 데이터는 목록에는 표시하되 지도에는 표시하지 않는다.
```

---

## 9-6. 테스트 결제 프롬프트

```text
Next.js 프로젝트에서 나이스페이먼츠 테스트 결제를 연결하고 싶어.

목표:
- 구독 화면에 구독 상품 카드를 표시한다.
- 사용자가 테스트 결제하기 버튼을 누르면 주문 정보를 생성한다.
- 나이스페이먼츠 테스트 결제창을 호출한다.
- 결제 성공 시 결제 성공 화면으로 이동한다.
- 결제 실패 시 결제 실패 화면으로 이동한다.
- 결제 성공 결과를 Supabase payments 테이블에 저장한다.

주의사항:
- Secret Key는 절대 브라우저에 노출하지 않는다.
- 실제 운영 결제가 아니라 테스트 결제 기준으로 구성한다.
- 실제 운영에서는 서버 검증이 필요하다는 설명을 포함해줘.
```

# 10. 수업에서 강조할 메시지

---

## 10-1. API는 기능을 빌려오는 통로입니다

로그인, 데이터, 지도, 결제는 모두 직접 만들기 어려운 기능입니다.

API를 사용하면 이미 만들어진 기능을 내 서비스에 연결할 수 있습니다.

---

## 10-2. API Key는 비밀번호처럼 다뤄야 합니다

API Key, Client Secret, Secret Key는 공개 저장소나 화면 캡처에 그대로 노출하면 안 됩니다.

---

## 10-3. AI에게는 “무엇을 연결할지”보다 “어떤 흐름으로 연결할지”를 알려줘야 합니다

나쁜 요청:

```text
결제 붙여줘.
```

좋은 요청:

```text
구독 상품을 선택하면 테스트 결제창을 호출하고,
성공 시 결제 결과를 저장한 뒤,
구독 상태를 활성화하는 흐름으로 만들어줘.
```

---

## 10-4. 완성보다 흐름을 이해하는 것이 먼저입니다

이번 프로젝트는 상용 서비스 완성이 아니라 API 연결 흐름을 이해하기 위한 수업용 뼈대입니다.

중요한 것은 아래 흐름을 이해하는 것입니다.

```text
사용자 행동
↓
API 요청
↓
응답 처리
↓
화면 표시
↓
서비스 기능으로 연결
```

# 12. 마지막 정리

이 프로젝트는 API 수업에서 사용할 통합 실습용 서비스입니다.

수강생은 이 프로젝트를 통해 API가 단순한 기술 용어가 아니라, 실제 서비스의 기능을 확장하는 연결 방식이라는 것을 이해할 수 있습니다.

이번 수업에서 중요한 것은 모든 API를 완벽하게 구현하는 것이 아닙니다.

중요한 것은 다음 흐름을 이해하는 것입니다.

```text
로그인으로 사용자를 확인하고,
공공 API로 데이터를 가져오고,
지도 API로 위치를 보여주고,
결제 API로 수익화 흐름을 체험한다.
```

이 감각을 잡으면 이후에는 다른 API도 같은 방식으로 접근할 수 있습니다.
